## 一、产品简介

TopoCode 是一款面向开发者的**源码架构分析与学习工具**。它能自动解析项目源码结构，生成多层级的组件依赖图、调用关系分析和架构分析报告，并支持 AI 辅助分析。

### 核心能力

| 能力 | 说明 |
|------|------|
| **多语言解析** | 基于 Tree-sitter AST 解析，支持 11 种语言（C、C++、C#、Java、JS、TS、Python、Go、Rust、PHP、Swift） |
| **架构分析** | 符号提取 → 调用图 → 依赖图 → Louvain 社区发现 |
| **AI 报告生成** | 基于 LLM 的组件分析、架构文档自动生成 |
| **可视化** | D3.js 力导向图、Mermaid/PlantUML 图表、Pixi.js 渲染 |
| **知识管理** (待开发) | 知识图谱、多维分类、TopoScript 动画演示 |
| **多项目** | 项目分组、任务管理、分析报告管理 |

## 二、安装与启动

### 环境要求

| 依赖 | 版本要求 |
|------|---------|
| Node.js | >= 18 |
| npm | >= 9 |
| Python | >= 3.10 |
| 系统 | Windows / macOS / Linux |

### 从源码安装

```bash
# 1. 克隆仓库
git clone https://github.com/topocode/topoone-ui.git
cd topoone-ui

# 2. 安装前端依赖
npm install

# 3. 创建 Python 虚拟环境并安装后端依赖
python3 -m venv .venv
source .venv/bin/activate  # Linux/macOS
pip install -r backend/requirements.txt

# 4. 构建并打包
npm run build
npm run package
```

### 开发模式运行

```bash
# 终端 1：启动前端开发服务器
npm run dev:vite

# 终端 2：启动 Python 后端
source .venv/bin/activate
python backend/main.py --http-port 3456

# 或者一键启动（Electron + 后端）
npm run dev
```

### 获取安装包

从各平台构建安装包：

```bash
npm run dist:linux   # Linux (.AppImage/.deb/.rpm)
npm run dist:mac     # macOS (.dmg/.zip)
npm run dist:win     # Windows (.exe NSIS 安装包)
```

## 五、代码解析

### 5.1 创建分析任务

![创建任务](images/screenshot-code-task-create.png)

*任务创建表单：选择范围、文件扩展名、排除目录、报告类型*

在 `TaskCreateForm` 中配置：

| 配置项 | 说明 |
|--------|------|
| 范围 | 选择要分析的文件/目录 |
| 扩展名 | 限定文件类型（如 `.py`, `.js`） |
| 排除目录 | 跳过无需分析的目录（如 `node_modules`） |
| 报告类型 | 选择要生成的分析报告类型 |

### 5.2 任务列表

![任务列表](images/screenshot-code-task-list.png)

*任务列表：展示所有分析任务及其状态、进度*

### 5.3 分析管道

任务创建后，系统自动执行 **6 步分析管道**：

```
Step 1: AST 解析      → Tree-sitter 解析语法树
Step 2: 符号提取      → 提取函数、类等全局符号
Step 3: 调用图提取    → 函数调用关系分析
Step 4: 依赖图提取    → 模块/文件依赖关系分析
Step 5: 社区发现      → Louvain 算法检测功能模块
Step 6: 结果汇总      → 汇总分析结果
```

### 5.4 任务详情

![任务详情](images/screenshot-code-task-detail.png)

*任务详情：AST 视图、调用链、依赖图、社区结果、执行日志*

任务完成后，可通过 `TaskDetailDialog` 查看：

- **AST 视图**：语法树结构
- **调用链**：函数间调用关系
- **依赖图**：模块/文件依赖关系
- **社区结果**：Louvain 算法划分的功能模块
- **执行日志**：每一步的执行日志

### 5.5 文件统计

![文件统计](images/screenshot-code-file-stats.png)

*文件统计面板：各语言行数分布、目录树筛选*

## 六、代码分析

### 6.1 报告生成管道

![报告生成管道](images/screenshot-analysis-report-pipeline.png)

*报告生成管道：多步骤 LLM 驱动的架构分析*

`ReportGenerationPipeline` 提供 LLM 驱动的报告生成：

```
Step 0: 文件摘要预处理  → 提取 README/依赖文件，生成文件摘要
Step 1-5: 架构分析     → LLM 逐层分析架构
社区 AI 分析           → 对每个社区进行 AI 解读
整体架构文档           → 生成全局架构报告
```

### 6.2 社区分析

![社区图谱](images/screenshot-analysis-community-graph.png)

*社区图谱：Louvain 算法划分的功能模块，支持多层级展开*

社区分析基于 Louvain 算法检测代码中的功能模块：

- **多层级**：支持 L0/L1 等多层级社区展开
- **边类型**：支持 `call`（调用）和 `include`（包含）两种关联
- **AI 解读**：选中社区后，LLM 自动生成功能描述和架构说明
- **图表渲染**：为每个社区生成 Mermaid/PlantUML 图

### 6.3 子文档查看

![子文档查看器](images/screenshot-analysis-subdoc-viewer.png)

*子文档查看器：Markdown 渲染、Mermaid/PlantUML 内联图表*

`SubDocViewer` 支持：

- Markdown 文档渲染
- Mermaid 图表内联渲染
- PlantUML 图表内联渲染
- 社区链接跳转
- 编辑/保存

### 6.4 Mermaid 图表渲染

![Mermaid 图表](images/screenshot-analysis-mermaid-diagram.png)

*Mermaid 图表：架构图、流程图内联渲染示例*

### 6.5 浏览器端查看

启动后端后，可通过浏览器访问 `http://localhost:3456` 查看：

- 架构文档
- 社区目录
- 代码查看

## 十、FAQ

### Q: 支持哪些编程语言？

目前支持 11 种语言：C、C++、C#、Java、JavaScript、TypeScript、Python、Go、Rust、PHP、Swift。更多语言持续扩展中。

### Q: 分析时后端未启动？

确保 Python 后端已启动。Status Bar 右下角显示后端连接状态（绿色=已连接，红色=异常）。

### Q: 如何配置 AI 模型？

进入设置页面 → 模型配置 → 添加模型。推荐使用 Ollama 或兼容 OpenAI API 的服务。

### Q: 分析报告如何导出？

报告生成后可通过 `SubDocViewer` 查看，也可通过浏览器访问 `http://localhost:3456` 查看在线版本。

### Q: 如何清除分析缓存？

在项目卡片上点击右键 → 清除缓存，可选择按表清除（base_node、graph_node 等）。

### Q: 报告支持导出 PDF 吗？

当前不直接支持 PDF 导出。可通过浏览器查看后，使用浏览器打印功能另存为 PDF。
