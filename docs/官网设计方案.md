# TopoCode 官网设计方案

> 风格: opencode 极简风格 | 目标: 产品展示 + 下载引导 + 文档入口

---

## 一、设计理念

### 核心原则

| 原则 | 说明 |
|------|------|
| **极简** | 去除冗余装饰，内容即界面 |
| **功能优先** | 每个元素都有明确目的 |
| **暗色基调** | 延续产品 Catppuccin Mocha 暗色主题，保持品牌一致性 |
| **代码气质** | 等宽字体、终端风格、代码截图展示 |

### 色彩体系（Catppuccin Mocha）

```
背景: #11111b (crust) / #1e1e2e (base)
卡片: #181825 (mantle) / #313244 (surface0)
文字: #cdd6f4 (text) / #a6adc8 (subtext0)
强调: #89b4fa (blue) / #cba6f7 (mauve) / #a6e3a1 (green)
边框: #45475a (surface1)
```

## 三、各区块详细设计

### 3.1 导航栏 (Nav)

```
[Logo: TopoCode]                           [Download] [GitHub] [Docs]
```

| 元素 | 说明 |
|------|------|
| Logo | 文字 Logo "TopoCode"，等宽字体加粗，accent 色 |
| Download | 主要 CTA 按钮，blue 背景 |
| GitHub | 文字链接，带 GitHub 图标 |
| Docs | 文字链接，指向使用文档 |

### 3.2 Hero 区

```
╔═══════════════════════════════════════════╗
║  TopoCode                                 ║
```

## 六、技术方案建议

### 方案一：静态站点生成器（推荐）

| 工具 | 说明 |
|------|------|
| **Astro** | 零 JS 默认输出，支持 Markdown 内容 |
| **Tailwind CSS** | 复用已有配置，保持一致风格 |
| **部署** | GitHub Pages / Vercel / Netlify |

**优势**: 加载快、维护简单、内容可放在项目 `website/` 目录中。

### 方案二：Vue SPA

| 工具 | 说明 |
|------|------|
| **Vue 3 + Vite** | 复用项目已有技术栈 |
| **Vue Router** | 多页支持 |
| **部署** | GitHub Pages |

**优势**: 与主项目技术栈一致，团队成员无需学习新工具。

### 建议目录结构

```
topoone-ui/
├── website/                  # 官网源码
│   ├── public/
│   │   ├── screenshots/      # 产品截图
│   │   └── icons/            # 图标
│   ├── src/
│   │   ├── components/       # Vue/Astro 组件
│   │   ├── sections/         # 页面 Section
│   │   └── styles/           # 全局样式
│   ├── index.html
│   └── package.json
├── docs/                     # 使用文档
│   └── 使用文档.md
└── README.md
```

## 七、内容策略

### 文案调性

| 维度 | 风格 |
|------|------|
| 语气 | 专业、直接、开发者导向 |
| 长度 | 简短有力，避免营销废话 |
| 示例 | "解析 11 种语言的源码结构" → 而非"支持多种主流编程语言" |

### 多语言

- **中文**: 主站语言
- **英文**: 第二版本（可后续增加）
